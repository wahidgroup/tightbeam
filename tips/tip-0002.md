## 1. Executive Summary

This document provides a comprehensive implementation plan for Layer 2 (ProcessSpec/CSP) and Layer 3 (FDR/Refinement) of the tightbeam testing framework. Layer 1 (AssertSpec) is complete and functional.

**Goals:**
1. Enable CSP-style state machine modeling with explicit external/internal event alphabets
2. Support nondeterministic choice points and labeled transition systems
3. Implement bounded refinement checking (traces, failures, divergence)
4. Provide optional CSPM export and FDR integration
5. Maintain backward compatibility with Layer 1

**Non-Goals:**
- Full FDR4 feature parity (we provide bounded approximation)
- Real-time or timing-dependent properties
- Model checking beyond refinement (e.g., CTL, LTL)

## 3. Layer 2: ProcessSpec Implementation

### 3.1 Core Concepts

**ProcessSpec** represents a labeled transition system (LTS) with:
- **States:** Discrete configuration points (e.g., `S0`, `S1`, `Waiting`, `Ready`)
- **Events:** Actions that cause transitions
	- **External:** Observable by environment (e.g., `"start"`, `"send"`, `"ack"`)
	- **Internal (τ):** Hidden implementation details (e.g., `"serialize"`, `"encrypt"`)
- **Transitions:** State × Event → State mappings
- **Nondeterministic Choice:** States with multiple possible transitions for the same event
- **Terminal States:** Accepting end states (success/failure)

**Key Insight:** AssertSpecs are deterministic linear traces. ProcessSpecs add branching and internal structure.

### 3.2 Macro Syntax: `tb_process_spec!`

Based on README Section 10.4:

```rust
tb_process_spec! {
	pub struct HandshakeSpec;
	
	// Event alphabets
	events {
		external { "start", "send", "ack", "fail" }
		internal { "serialize", "encrypt", "queue", "dispatch" }
	}
	
	// State transition table
	states {
		S0  => { "start" => S1 }
		S1  => { "serialize" => S1s, "queue" => S1q }
		S1s => { "encrypt" => S1e }
		S1e => { "send" => S2 }
		S1q => { "dispatch" => S1d }
		S1d => { "send" => S2 }
		S2  => { "ack" => S3, "fail" => S3f }
		S3  => {}
		S3f => {}
	}
	
	// Terminal/accepting states
	terminal { S3, S3f }
	
	// Nondeterministic choice points
	nondeterministic { S1 }
	
	// Optional metadata
	annotations {
		description: "Queued or direct send handshake"
	}
}
```

### 3.3 Internal Representation

```rust
// tightbeam/src/testing/process.rs (NEW FILE)

#[cfg(feature = "testing-csp")]
pub mod process {
	use std::collections::{HashMap, HashSet};
	
	/// Event classification
	#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
	pub enum EventClass {
		External,
		Internal,
	}
	
	/// State identifier (typically enum or string)
	pub type StateId = &'static str;
	
	/// Event name
	pub type EventName = &'static str;
	
	/// Transition: (from_state, event) -> to_state
	pub type Transition = (StateId, EventName, StateId);
	
	/// ProcessSpec trait - generated by macro
	pub trait ProcessSpec: Send + Sync + 'static {
		/// Get process name
		fn name(&self) -> &'static str;
		
		/// Get external alphabet
		fn external_alphabet(&self) -> &HashSet<EventName>;
		
		/// Get internal alphabet (tau events)
		fn internal_alphabet(&self) -> &HashSet<EventName>;
		
		/// Get initial state
		fn initial_state(&self) -> StateId;
		
		/// Get terminal states
		fn terminal_states(&self) -> &HashSet<StateId>;
		
		/// Get nondeterministic choice states
		fn nondeterministic_states(&self) -> &HashSet<StateId>;
		
		/// Execute transition: (state, event) -> Option<Vec<StateId>>
		/// Returns None if transition not defined
		/// Returns Some(vec) with possible next states (1+ for nondeterminism)
		fn step(&self, state: StateId, event: EventName) -> Option<Vec<StateId>>;
		
		/// Get enabled events at state
		fn enabled(&self, state: StateId) -> HashSet<EventName>;
		
		/// Check if event is internal (tau)
		fn is_internal(&self, event: EventName) -> bool {
			self.internal_alphabet().contains(event)
		}
		
		/// Get all transitions
		fn transitions(&self) -> &[Transition];
		
		/// Get process hash (deterministic)
		fn spec_hash(&self) -> [u8; 32];
	}
	
	/// Concrete implementation generated by macro
	pub struct ProcessSpecImpl {
		pub name: &'static str,
		pub external: HashSet<EventName>,
		pub internal: HashSet<EventName>,
		pub initial: StateId,
		pub terminal: HashSet<StateId>,
		pub nondeterministic: HashSet<StateId>,
		pub transition_map: HashMap<(StateId, EventName), Vec<StateId>>,
		pub transitions: Vec<Transition>,
		pub spec_hash: [u8; 32],
	}
	
	impl ProcessSpec for ProcessSpecImpl {
		fn name(&self) -> &'static str { self.name }
		fn external_alphabet(&self) -> &HashSet<EventName> { &self.external }
		fn internal_alphabet(&self) -> &HashSet<EventName> { &self.internal }
		fn initial_state(&self) -> StateId { self.initial }
		fn terminal_states(&self) -> &HashSet<StateId> { &self.terminal }
		fn nondeterministic_states(&self) -> &HashSet<StateId> { &self.nondeterministic }
		
		fn step(&self, state: StateId, event: EventName) -> Option<Vec<StateId>> {
			self.transition_map.get(&(state, event)).cloned()
		}
		
		fn enabled(&self, state: StateId) -> HashSet<EventName> {
			self.transition_map
				.keys()
				.filter(|(s, _)| *s == state)
				.map(|(_, e)| *e)
				.collect()
		}
		
		fn transitions(&self) -> &[Transition] { &self.transitions }
		fn spec_hash(&self) -> [u8; 32] { self.spec_hash }
	}
}
```

### 3.4 Macro Implementation Strategy

```rust
// tightbeam/src/testing/macros.rs - Add new section

#[cfg(feature = "testing-csp")]
#[macro_export]
macro_rules! tb_process_spec {
	(
		pub struct $name:ident;
		events {
			external { $($ext_event:literal),* $(,)? }
			internal { $($int_event:literal),* $(,)? }
		}
		states {
			$(
				$state:ident => {
					$( $event:literal => $next:ident ),* $(,)?
				}
			)*
		}
		terminal { $($term:ident),* $(,)? }
		$(nondeterministic { $($nondet:ident),* $(,)? })?
		$(annotations { $($key:ident : $val:expr),* $(,)? })?
	) => {
		pub struct $name;
		
		impl $name {
			pub fn instance() -> &'static $crate::testing::process::ProcessSpecImpl {
				static SPEC: ::std::sync::OnceLock<$crate::testing::process::ProcessSpecImpl> = ::std::sync::OnceLock::new();
				SPEC.get_or_init(|| {
					use $crate::testing::process::*;
					use std::collections::{HashMap, HashSet};
					
					// Build alphabets
					let external: HashSet<EventName> = vec![$($ext_event),*].into_iter().collect();
					let internal: HashSet<EventName> = vec![$($int_event),*].into_iter().collect();
					
					// Build terminal set
					let terminal: HashSet<StateId> = vec![$(stringify!($term)),*].into_iter().collect();
					
					// Build nondeterministic set
					#[allow(unused_mut)]
					let mut nondeterministic: HashSet<StateId> = HashSet::new();
					$(
						nondeterministic.extend(vec![$(stringify!($nondet)),*]);
					)?
					
					// Build transition map
					let mut transition_map: HashMap<(StateId, EventName), Vec<StateId>> = HashMap::new();
					let mut transitions: Vec<Transition> = Vec::new();
					
					$(
						$(
							let from = stringify!($state);
							let event = $event;
							let to = stringify!($next);
							
							transition_map
								.entry((from, event))
								.or_insert_with(Vec::new)
								.push(to);
							
							transitions.push((from, event, to));
						)*
					)*
					
					// Compute spec hash
					let spec_hash = Self::compute_hash(
						stringify!($name),
						&external,
						&internal,
						&transitions,
						&terminal,
						&nondeterministic,
					);
					
					ProcessSpecImpl {
						name: stringify!($name),
						external,
						internal,
						initial: "S0", // Convention: first state is initial
						terminal,
						nondeterministic,
						transition_map,
						transitions,
						spec_hash,
					}
				})
			}
			
			fn compute_hash(
				name: &str,
				external: &HashSet<EventName>,
				internal: &HashSet<EventName>,
				transitions: &[Transition],
				terminal: &HashSet<StateId>,
				nondeterministic: &HashSet<StateId>,
			) -> [u8; 32] {
				use sha3::{Digest, Sha3_256};
				
				let mut hasher = Sha3_256::new();
				
				// Domain tag
				hasher.update(b"TBPS"); // TightBeam Process Spec
				
				// Name
				hasher.update(name.as_bytes());
				
				// External alphabet (sorted)
				let mut ext_sorted: Vec<_> = external.iter().collect();
				ext_sorted.sort();
				for e in ext_sorted {
					hasher.update(b"EXT:");
					hasher.update(e.as_bytes());
				}
				
				// Internal alphabet (sorted)
				let mut int_sorted: Vec<_> = internal.iter().collect();
				int_sorted.sort();
				for e in int_sorted {
					hasher.update(b"INT:");
					hasher.update(e.as_bytes());
				}
				
				// Transitions (sorted)
				let mut trans_sorted = transitions.to_vec();
				trans_sorted.sort();
				for (from, event, to) in trans_sorted {
					hasher.update(b"TRANS:");
					hasher.update(from.as_bytes());
					hasher.update(b"->");
					hasher.update(event.as_bytes());
					hasher.update(b"->");
					hasher.update(to.as_bytes());
				}
				
				// Terminal states (sorted)
				let mut term_sorted: Vec<_> = terminal.iter().collect();
				term_sorted.sort();
				for s in term_sorted {
					hasher.update(b"TERM:");
					hasher.update(s.as_bytes());
				}
				
				// Nondeterministic states (sorted)
				let mut nondet_sorted: Vec<_> = nondeterministic.iter().collect();
				nondet_sorted.sort();
				for s in nondet_sorted {
					hasher.update(b"NONDET:");
					hasher.update(s.as_bytes());
				}
				
				hasher.finalize().into()
			}
		}
		
		impl $crate::testing::process::ProcessSpec for $name {
			fn name(&self) -> &'static str {
				Self::instance().name()
			}
			
			fn external_alphabet(&self) -> &HashSet<EventName> {
				Self::instance().external_alphabet()
			}
			
			fn internal_alphabet(&self) -> &HashSet<EventName> {
				Self::instance().internal_alphabet()
			}
			
			fn initial_state(&self) -> StateId {
				Self::instance().initial_state()
			}
			
			fn terminal_states(&self) -> &HashSet<StateId> {
				Self::instance().terminal_states()
			}
			
			fn nondeterministic_states(&self) -> &HashSet<StateId> {
				Self::instance().nondeterministic_states()
			}
			
			fn step(&self, state: StateId, event: EventName) -> Option<Vec<StateId>> {
				Self::instance().step(state, event)
			}
			
			fn enabled(&self, state: StateId) -> HashSet<EventName> {
				Self::instance().enabled(state)
			}
			
			fn transitions(&self) -> &[Transition] {
				Self::instance().transitions()
			}
			
			fn spec_hash(&self) -> [u8; 32] {
				Self::instance().spec_hash()
			}
		}
	};
}
```

### 3.5 Promotion: AssertSpec → ProcessSpec

Allow automatic generation of linear LTS from deterministic AssertSpec:

```rust
// tightbeam/src/testing/process.rs

#[cfg(feature = "testing-csp")]
pub fn promote_assert_to_process<A: AssertSpec>(
	assert_spec: &A,
	name: &'static str,
) -> ProcessSpecImpl {
	use std::collections::{HashMap, HashSet};
	
	let assertions = assert_spec.assertions();
	let mut external = HashSet::new();
	let mut transitions = Vec::new();
	let mut transition_map = HashMap::new();
	
	// Create linear state chain
	let mut state_id = 0;
	let initial = format!("S{}", state_id);
	
	for (idx, assertion) in assertions.iter().enumerate() {
		let from = format!("S{}", idx);
		let to = format!("S{}", idx + 1);
		let event = assertion.label;
		
		external.insert(event);
		transitions.push((from.leak() as &str, event, to.leak() as &str));
		transition_map
			.entry((from.leak() as &str, event))
			.or_insert_with(Vec::new)
			.push(to.leak() as &str);
	}
	
	let terminal_state = format!("S{}", assertions.len());
	let mut terminal = HashSet::new();
	terminal.insert(terminal_state.leak() as &str);
	
	let spec_hash = compute_promotion_hash(name, &transitions);
	
	ProcessSpecImpl {
		name,
		external,
		internal: HashSet::new(),
		initial: initial.leak(),
		terminal,
		nondeterministic: HashSet::new(),
		transition_map,
		transitions,
		spec_hash,
	}
}
```

## 4. Layer 3: FDR/Refinement Implementation

### 4.1 Refinement Concepts

**Refinement** checks whether an implementation satisfies a specification:
- **Trace Refinement (⊑T):** Every trace of Impl is allowed by Spec
- **Failures Refinement (⊑F):** Every (trace, refusal-set) of Impl is allowed by Spec
- **Failures-Divergence (⊑FD):** Failures + divergence freedom

**Our Approach:** Bounded exploration with multiple schedulers/seeds to approximate these properties.

### 4.2 Exploration Profile

```rust
pub mod refinement {
	use crate::testing::process::ProcessSpec;
	use std::collections::{HashMap, HashSet};
	
	/// Exploration strategy
	#[derive(Debug, Clone, Copy)]
	pub enum ExplorationMode {
		/// DFS up to max_depth
		ExhaustiveBounded,
		/// Random walk
		RandomSampling,
		/// Breadth-first
		BFS,
	}
	
	/// Exploration profile
	#[derive(Debug, Clone)]
	pub struct TBProfile {
		pub schedulers: usize,           // Parallel exploration threads
		pub seeds: usize,                // Random seeds for nondeterminism
		pub max_depth: usize,            // Max trace length
		pub max_internal_run: usize,     // Max consecutive tau steps (divergence threshold)
		pub timeout_ms: u64,             // Total timeout
		pub mode: ExplorationMode,
	}
	
	impl Default for TBProfile {
		fn default() -> Self {
			Self {
				schedulers: 4,
				seeds: 64,
				max_depth: 128,
				max_internal_run: 32,
				timeout_ms: 5000,
				mode: ExplorationMode::ExhaustiveBounded,
			}
		}
	}
	
	/// Refinement verdict

	pub struct Verdict {
		pub trace_refines: bool,
		pub failures_refines: bool,
		pub divergence_free: bool,
		pub deterministic: bool,

		pub unexpected_traces: Vec<Vec<&'static str>>,
		pub refusal_diffs: Vec<RefusalDiff>,
```

### 4.4 CSPM Export

```rust
#[cfg(feature = "testing-fdr")]
pub fn export_cspm<S: ProcessSpec>(spec: &S, output_path: &std::path::Path) -> std::io::Result<()> {
	use std::io::Write;
	
	let mut file = std::fs::File::create(output_path)?;
	
	// Header
	writeln!(file, "-- Generated by tightbeam")?;
	writeln!(file, "-- Process: {}", spec.name())?;
	writeln!(file)?;
	
	// Alphabets
	writeln!(file, "-- External events")?;
	write!(file, "channel ")?;
	let ext: Vec<_> = spec.external_alphabet().iter().collect();
	for (idx, event) in ext.iter().enumerate() {
		write!(file, "{}", event)?;
		if idx < ext.len() - 1 {
			write!(file, ", ")?;
		}
	}
	writeln!(file)?;
	writeln!(file)?;
	
	// Process definition (simplified - full CSP syntax is complex)
	writeln!(file, "{} = {}", spec.name(), spec.initial_state())?;
	writeln!(file)?;
	
	// State processes
	let mut state_map: HashMap<&str, Vec<_>> = HashMap::new();
	for (from, event, to) in spec.transitions() {
		state_map.entry(from).or_insert_with(Vec::new).push((event, to));
	}
	
	for (state, transitions) in state_map {
		if spec.terminal_states().contains(state) {
			writeln!(file, "{} = SKIP", state)?;
		} else {
			write!(file, "{} = ", state)?;
			for (idx, (event, to)) in transitions.iter().enumerate() {
				write!(file, "{} -> {}", event, to)?;
				if idx < transitions.len() - 1 {
					if spec.nondeterministic_states().contains(state) {
						write!(file, " |~| ")?; // Internal choice
					} else {
						write!(file, " [] ")?; // External choice
					}
				}
			}
			writeln!(file)?;
		}
	}
	
	Ok(())
}
```

## 7. Open Questions

1. **State Representation:** Should we use `&'static str` or enum-based states?
	- **Decision:** Start with `&'static str` for flexibility, optimize later if needed

2. **Nondeterminism Resolution:** How do we choose among multiple enabled transitions?
	- **Decision:** Use seeded RNG in Layer 3, expose choice points in Layer 2

3. **Internal Event Hiding:** How do we project traces to external alphabet?
	- **Decision:** Filter internal events when comparing against spec

4. **CSPM Syntax:** Full CSP or simplified subset?
	- **Decision:** Simplified subset initially, expand as needed

5. **FDR Integration:** Direct API or file-based?
	- **Decision:** File-based initially (CSPM export + result parsing)

6. **Performance:** How large can state spaces be?
	- **Decision:** Target 10K states, 100K transitions as reasonable upper bound
